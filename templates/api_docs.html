Runtime prompt provider

Compiled Prompts API

Load active prompts from Prompt Admin with hook placeholders already resolved before n8n calls an LLM.

Try JSON
JSON endpoint

GET /api/prompts/compiled

The endpoint compiles prompts through the same hook resolver used by the preview page and returns JSON for workflow runtime usage.

Runtime field compiled_prompt
Returned records active prompts only
Hook behavior active hooks inserted

Host URL

API contract

Prompt Admin owns prompt compilation. n8n should not query PostgreSQL directly.

  • PostgreSQL is the source of truth for prompts, hooks, and families.
  • This runtime endpoint returns compiled prompt text for active prompts only.
  • Inactive and soft-deleted prompts are intentionally excluded.
  • Prompt-level description is intentionally not returned.
  • Hook description is returned only as resolved-hook metadata.
  • Prompt families are UI grouping metadata, not a runtime selection API.
  • n8n should select prompts by stable prompt_key or category.

Request modes

Choose category loading, key loading, or a combined filter.

GET

By category

Load all active prompts from one category.

http://prompt-admin:8090/api/prompts/compiled?category=image_generation
GET

By repeated keys

Load several specific prompts by passing multiple key parameters.

http://prompt-admin:8090/api/prompts/compiled?key=script_writer&key=tag_retrieval_planner
GET

By comma-separated keys

Load several specific prompts through one keys parameter.

http://prompt-admin:8090/api/prompts/compiled?keys=script_writer,tag_retrieval_planner
GET

Category + keys

Return the intersection: active prompts in the category and matching the requested keys.

http://prompt-admin:8090/api/prompts/compiled?category=image_generation&keys=script_writer,tag_retrieval_planner

Workflow shape

  1. HTTP RequestLoad compiled prompts from Prompt Admin.
  2. Code nodeConvert prompts[] into a prompt map.
  3. LLM nodeUse prompt_map.script_writer as the system prompt.

n8n Code node

Convert the returned prompt array into a key-value map.

const promptMap = {};

for (const prompt of $$json.prompts || []) {
  promptMap[prompt.prompt_key] = prompt.compiled_prompt;
}

return [
  {
    json: {
      ...$$json,
      prompt_map: promptMap,
    },
  }
];

Use later in n8n

{{$$json.prompt_map.script_writer}}

Prompt record fields

Field Meaning
prompt_keyStable prompt identifier used by n8n.
categoryPrompt-level category.
is_activeAlways true for returned prompts.
updated_atPrompt update timestamp, or null.
raw_promptOriginal prompt text from PostgreSQL.
compiled_promptFinal prompt text with hooks inserted.
detected_groups[]Hook groups detected in the raw prompt.
resolved_hooks[]Hooks inserted into the compiled prompt.
unresolved_groups[]Hook placeholders without matching active hooks.

Resolved hook fields

Field Meaning
hook_keyStable hook identifier.
hook_groupResolved placeholder group.
descriptionHuman-facing hook note.
categoryHook category metadata.
priorityHook ordering value. Lower values are inserted first.
is_activeWhether the hook was active when resolved.

Runtime behavior

  • At least one of category, key, or keys is required.
  • Only active prompts are returned.
  • Soft-deleted prompts are not returned.
  • Only active hooks are inserted.
  • Duplicate requested keys are ignored while preserving the first occurrence.
  • Requested key order is preserved in prompts[].
  • missing_keys reports requested keys that were not returned.
  • missing_keys is empty for category-only requests.
  • The main field for n8n is compiled_prompt.
